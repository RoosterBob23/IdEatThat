I want to create a card game web app.  

I want a user to be able to click on a link to start a new game.  The user (Game Master) will enter their name and be presented with a link to the game that he can share with the other players.   If a game is created and it is not started within one hour, cancel the game. When a player logs in, they provide a name.  The Game Master needs to see who has logged in.  Once all the players log in, the Game Master starts the game.

A summary of game play.  Each round a player will become the Critic.  

Players will have an opportunity to play Theme cards, starting with the Critic.  Only one Theme card can be played in any round.  

Users will select cards from their hand to submit to the critic.  Each player should only be able to see their hand while selecting cards. After cards are submitted, players have an opportunity to play cards that sabotage other player(s).  

Once all cards are played, the cards are revealed to all players, grouped by player, but the player who submitted the cards is not visible.  The Critic looks at the cards submitted by each player and declares a winner by clicking on a group of cards.  Once the Critic clicks on a set of cards, the name of the player who submitted each group of cards should be displayed to identify which player submitted which cards.

The Critic clicks the Next Round button.  The cards played are discarded and each player draws cards to bring their hand up to five cards.  The next player becomes the Critic, and another round starts.  The first player to win five rounds, wins the game.  All players should be able to see how many rounds each player has won.

Once the game is over, display a leader board ranking each player by the number of games won.  Allow the Game Master to start another game.  Once a game has had no activity for one hour, delete the game.

There can be multiple games going on at the same time.
